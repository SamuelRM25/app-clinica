# CONTEXTO DEL PROYECTO - Centro Médico RS (app-clinica)

## RESUMEN EJECUTIVO

Sistema de gestión hospitalaria completo desarrollado en PHP puro para el "Centro Médico RS". Es una aplicación web multi-tenant (soporta múltiples hospitales) con sistema de suscripción y módulos activables por hospital. Permite administrar pacientes, citas, facturación, farmacia, laboratorio, hospitalización, inventario, compras y reportes.

---

## STACK TECNOLÓGICO

- **Backend:** PHP puro (sin framework)
- **Base de datos:** MySQL (alojada en CleverCloud)
- **Frontend:** HTML, CSS custom con variables, JavaScript vanilla
- **Librerías externas (CDN):**
  - Bootstrap 5.3.x (UI framework)
  - Bootstrap Icons 1.10.x (iconos)
  - SweetAlert2 11 (alertas y confirmaciones)
  - FullCalendar 5.11.3 (calendario de citas)
  - jQuery 3.6.0 + Select2 4.1.0 (búsquedas AJAX)
  - Google Fonts - Inter (tipografía)
- **Servidor:** Apache (con .htaccess para configuración de sesiones)

El sistema usa un modelo de **suscripción con módulos activables**. Los módulos se almacenan como JSON en `hospitales.modulos_activos`. Cada módulo se verifica con `check_module_access('module_name')`.

| Módulo | Label | Descripción |
|--------|-------|-------------|
| `core` | Core | Siempre activo - Pacientes, Citas, Cobros |
| `pharmacy` | Farmacia | Punto de venta con reservas de inventario |
| `hospitalization` | Hospitalización | Ingreso de pacientes, gestión de camas |
| `laboratory` | Laboratorio | Órdenes, catálogos, procesamiento de muestras, validación |
| `inventory` | Inventario | Medicamentos, insumos, vencimientos, órdenes de compra |
| `imaging` | Imagenología | Procedimientos menores + Exámenes |
| `purchases` | Compras | Órdenes de compra, pagos, proveedores |
| `sales` | Ventas | Reportes de ventas, cortes de turno |
| `finances` | Finanzas | Cobros y facturación |
| `reports` | Reportes | Exportaciones (jornada, transferencias, ventas, labs) |

## AUTENTICACIÓN Y AUTORIZACIÓN

### Flujo de Login
1. Usuario ingresa en `index.php` → envía formulario a `php/auth/login.php`
2. Se consulta `usuarios` JOIN `hospitales` por `codigo_hospital`
3. Comparación de contraseña en **texto plano** (sin hash)
4. Si éxito, se almacena en `$_SESSION`: `user_id`, `nombre`, `id_hospital`, `apellido`, `clinica`, `especialidad`, `tipoUsuario`, `usuario`
5. Se carga configuración del hospital (módulos, suscripción, tema)
6. Redirección a `php/dashboard/index.php`

### Niveles de Usuario
- **admin** - Acceso total
- **doc** - Doctor
- **user** - Usuario regular

### Seguridad Multi-tenant
- `verify_session()` en `includes/functions.php` - verifica sesión activa
- `check_module_access()` en `includes/module_guard.php` - verifica sesión + módulo activo
- `hospital_id()` - retorna ID del hospital actual para filtrado SQL
- Validación cruzada de hospital en sesión (destruye sesión si hay mismatch)
- Filtrado SQL por hospital en todas las consultas

### Super Admin
- Panel separado en `admin.php`
- Credenciales hardcodeadas: `superadmin` / `root`
- Gestiona todos los hospitales, suscripciones, solicitudes de módulos y usuarios

## FUNCIONES CLAVE

### `includes/functions.php`
- `sanitize_input($data)` - trim/stripslashes/htmlspecialchars
- `verify_session()` - verificación de sesión con redirect
- `time_ago($datetime)` - formato de tiempo relativo en español
- `output_keep_alive_script()` - script auto-inyectado para mantener sesión activa (ping cada 5 min)
- `compressImage($sourcePath, $destinationPath, $quality)` - compresión de imágenes JPG/PNG

### `includes/module_guard.php`
- `check_module_access(string $module_name)` - doble verificación: sesión + suscripción de módulo
- `hospital_id()` - retorna `$_SESSION['id_hospital']` como int

### `includes/multitenant.php`
- `get_hospital_config($conn, $hospital_id)` - obtiene config del hospital, sincroniza a sesión
- `is_module_active($module_name)` - verifica si módulo está en lista activa
- `check_subscription_status()` - valida suscripción (Activo/Vencido/De por vida)
- `get_hospital_filter($alias)` - genera cláusula WHERE SQL para aislamiento por hospital
- `apply_hospital_theme()` - inyecta CSS del tema según preferencia del hospital

### `assets/js/human_body_map.js`
- Objeto `HumanBodyMap` con datos SVG de 28+ regiones anatómicas (frontal/posterior)
- Selector interactivo de zonas de dolor con feedback visual
- Output: nombres de regiones separados por coma en input hidden

### `assets/js/validate_result.js`
- `validateResult(input, min, max, statusId)` - valida resultados de laboratorio contra rangos de referencia

### `assets/js/security.js`
- Bloqueo de clic derecho, F12, Ctrl+Shift+I/J/C, Ctrl+U, Cmd+Option+I
- Detección de DevTools via timing de `debugger` y diferencia de tamaño de ventana
- Deshabilita selección de texto y arrastre de imágenes

## MEDIDAS DE SEGURIDAD

### Implementadas
1. **Prepared Statements** - Todas las consultas SQL usan PDO prepared statements (previene SQL injection)
2. **Input Sanitization** - `sanitize_input()` (trim, stripslashes, htmlspecialchars)
3. **Autenticación por sesión** - Todas las páginas verifican `$_SESSION['user_id']`
4. **Control de acceso a módulos** - `check_module_access()` protege cada módulo
5. **Aislamiento multi-tenant** - Filtrado por hospital ID en todas las consultas
6. **Validación de suscripción** - Módulos bloqueados para suscripciones inactivas
7. **Protección DevTools** - Bloqueo de clic derecho, atajos de teclado, detección de DevTools
8. **Selección de texto deshabilitada** - CSS `user-select: none` excepto en inputs
9. **Keep-alive de sesión** - Ping automático cada 5 minutos
10. **Corte de turno protegido** - Requiere código de seguridad ('cmhs')
11. **Error logging** - Errores de BD se loguean en lugar de mostrarse

### NO Implementadas (Preocupaciones de Seguridad)
1. **Password Hashing** - Contraseñas en **texto plano** (sin password_hash/password_verify)
2. **CSRF Protection** - Sin tokens CSRF en formularios
3. **XSS Prevention** - `htmlspecialchars()` usado pero no consistentemente
4. **Rate Limiting** - Sin throttling de intentos de login
5. **Session Fixation** - Sin regeneración de sesión post-login
6. **HTTPS Enforcement** - Sin redirect HTTP→HTTPS
7. **File Upload Validation** - Archivos de resultados de laboratorio sin validación
8. **Admin Credentials Hardcoded** - Credenciales de super admin en código fuente
9. **Database Credentials Exposed** - Credenciales de BD en texto plano en `config/database.php`
10. **No Input Validation** - Sin validación de tipo/rango más allá de sanitización básica

## CARACTERÍSTICAS DESTACADAS

1. **Sistema de reservas de inventario** - Durante una venta en farmacia, los items se reservan temporalmente (60 min) para evitar conflictos de stock
2. **Mapa corporal interactivo** - SVG con 28+ regiones anatómicas clickeables para registrar puntos de dolor
3. **Corte de turno** - Resumen financiero del turno con código de seguridad
4. **Widgets configurables** - Dashboard con widgets que se pueden mostrar/ocultar por hospital
5. **Precios dinámicos por doctor** - Reglas de precio según hora del día y día de la semana
6. **Multi-tema por hospital** - Cada hospital puede tener su tema de color personalizado
7. **Sistema de suscripción** - Módulos activables/desactivables con fechas de vencimiento
8. **Detección de pacientes duplicados** - Confirmación antes de crear paciente con datos similares
9. **Validación de resultados de laboratorio** - Rangos mín/máx configurables por prueba
10. **Exportaciones PDF** - Reportes de inventario, compras, ventas, jornada

## NOTAS IMPORTANTES PARA DESARROLLO FUTURO

- El sistema está diseñado para ser **multi-tenant**, pero actualmente opera principalmente con el hospital 'RS001'
- Las contraseñas están en texto plano - **prioridad crítica** implementar hashing
- No hay framework MVC - cada módulo es un conjunto de archivos PHP independientes
- Las consultas SQL están embebidas directamente en los archivos PHP
- El sistema usa sesiones PHP nativas sin abstracción
- Los temas se aplican via CSS variables y se almacenan por hospital
- El sistema de módulos es JSON-based en la columna `modulos_activos` de la tabla `hospitales`
